Lesson 01

Introduction to JavaScript

What is JavaScript and why does it matter?

What is JavaScript?

JavaScript is a programming language that makes websites interactive. While HTML provides structure and CSS provides styling, JavaScript provides behavior — it's what makes buttons respond to clicks, forms validate input, and content update without reloading the page.

JavaScript runs in the browser (client-side) and can also run on servers (Node.js). It is one of the three core technologies of the web, alongside HTML and CSS.

A Brief History

JavaScript was created by Brendan Eich in 1995 in just 10 days while working at Netscape. Despite its name, JavaScript has no direct relation to Java. The language has evolved significantly, with modern standards maintained by ECMAScript (ES6, ES2015, ES2020, etc.).

What Can JavaScript Do?

  • Respond to user interactions (clicks, typing, scrolling)
  • Manipulate HTML and CSS dynamically
  • Validate form data before submission
  • Create animations and visual effects
  • Fetch data from servers without page reload (AJAX)
  • Build complete web applications (React, Vue, Angular)
  • Create mobile apps (React Native) and desktop apps (Electron)

Where Does JavaScript Code Go?

JavaScript can be added to HTML in three ways:

1. Inline JavaScript

<button onclick="alert('Hello!')">Click Me</button>

2. Internal JavaScript

<script>
  console.log("Hello from JavaScript!");
</script>

3. External JavaScript (Recommended)

<script src="script.js"></script>
✅
Best Practice

Always place your <script> tags just before the closing </body> tag, or use the defer attribute. This ensures the HTML loads before JavaScript tries to manipulate it.

Your First JavaScript Program

Create an HTML file with the following:

<!DOCTYPE html>
<html>
  <head>
    <title>My First JavaScript</title>
  </head>
  <body>
    <h1>Hello JavaScript!</h1>
    
    <script>
      alert("Welcome to JavaScript!");
      console.log("Check the browser console!");
    </script>
  </body>
</html>

Lesson 02

JavaScript Language and Syntax

Understanding the rules and structure

JavaScript Statements

A statement is a single instruction that tells the computer to do something. Statements end with a semicolon ; (optional but recommended). 

console.log("This is a statement");
alert("This is another statement");
let x = 5; // Variable declaration statement

Comments

Comments are notes for developers that JavaScript ignores.

// This is a single-line comment

/* 
  This is a 
  multi-line comment
*/

let age = 25; // Inline comment after code

Case Sensitivity

JavaScript is case-sensitive. myVariable, MyVariable, and MYVARIABLE are three different identifiers.

let name = "John";
let Name = "Jane";  // Different variable!
console.log(name);  // "John"
console.log(Name);  // "Jane"

Naming Conventions

Follow these rules when naming variables and functions:

  • Must start with a letter, underscore (_), or dollar sign ($)
  • Cannot start with a number
  • Cannot use reserved keywords (let, if, function, etc.)
  • Use camelCase for variables: firstName, totalPrice
  • Use PascalCase for classes: UserProfile, ShoppingCart
  • Use UPPERCASE for constants: MAX_SIZE, API_KEY
// ✅ Valid names
let firstName = "Amara";
let _privateVar = 10;
let $element = document.querySelector("div");
let user2 = "Chidi";

// ❌ Invalid names
let 2users = 5;      // Cannot start with number
let my-name = "Ada"; // Hyphens not allowed
let let = 10;        // 'let' is a reserved keyword

JavaScript Keywords

These words are reserved and cannot be used as variable names:

break, case, catch, class, const, continue, debugger, 
default, delete, do, else, export, extends, finally, 
for, function, if, import, in, instanceof, let, new, 
return, super, switch, this, throw, try, typeof, var, 
void, while, with, yield

Whitespace and Formatting

JavaScript ignores extra spaces, tabs, and line breaks. Use them to make code readable.

// Both are valid, but the second is more readable
let x=5;let y=10;let z=x+y;

let x = 5;
let y = 10;
let z = x + y;
💡
Good to Know

Modern JavaScript uses let and const instead of var. We'll cover the differences in the next chapter.

Output & Input Methods

alert, prompt, and console.log — your first tools

Three Ways to Communicate

Before building web pages, you need to see results and get user input. JavaScript gives you three essential tools:

MethodPurposeReturns
alert()Show a popup messageundefined
prompt()Ask user to type somethingString or null
console.log()Print to Developer Consoleundefined

alert() — Show a Message

alert("Welcome to JavaScript class!");
alert("Your score is: " + 95);

Opens a blocking popup. The user must click OK before the page continues.

prompt() — Get User Input

let name = prompt("What is your name?");
alert("Hello, " + name + "!");

let age = prompt("How old are you?");

console.log("Hello, World!");
console.log("Name:", name, "Age:", age);
console.log(10 + 5);
console.log({ name: "Ada", score: 90 });

Open the console: F12 → Console tab (or Right-click → Inspect → Console). This is where developers debug code daily.

Lesson 04

Data & Types

Primitive and non-primitive data types

What Is Data?

Data is any value your program works with — a name, age, price, or login status. Every value in JavaScript has a type that tells the engine how to store and use it.

Two Categories

CategoryTypesStored
PrimitiveString, Number, Boolean, null, undefined, Symbol, BigIntBy value
Non-PrimitiveObject, Array, FunctionBy reference

String

Text wrapped in quotes. Three ways to create strings:

let a = "double quotes";
let b = 'single quotes';
let c = `backticks (template literals)`;

// Concatenation
let first = "Ada";
let last = "Okafor";
let full = first + " " + last;           // "Ada Okafor"
let greeting = `Hello, ${first}!`;      // "Hello, Ada!"

// Length and index (starts at 0)
console.log(full.length);   // 10
console.log(full[0]);       // "A"
console.log(full[4]);       // "O"

// Escape characters
let quote = "She said, \"Hello!\"";   // quotes inside
let path = "C:\\Users\\Desktop";      // backslash
let lines = "Line 1\nLine 2";         // new line
let tab = "Name\tAge";                // tab

Number

let age = 25;
let price = 99.99;

console.log(10 + 3);   // 13  addition
console.log(10 - 3);   // 7   subtraction
console.log(10 * 3);   // 30  multiplication
console.log(10 / 3);   // 3.333... division
console.log(10 % 3);   // 1   modulus (remainder)

// Modulus use cases
console.log(10 % 2 === 0);  // true → even
console.log(7 % 2 === 1);   // true → odd

Boolean

let isLoggedIn = true;
let hasPermission = false;
let isAdult = age >= 18;  // evaluates to true or false

null and undefined

let selectedUser = null;    // intentionally empty
let username;               // undefined — not assigned yet
console.log(username);      // undefined

null = you set it empty. undefined = JavaScript hasn't given it a value yet.

Symbol & BigInt (Brief)

// Symbol — unique identifier (advanced use)
const id = Symbol("id");
const id2 = Symbol("id");
console.log(id === id2);  // false

// BigInt — very large integers (note the 'n' suffix)
const big = 9007199254740991n;
console.log(big + 1n);

Non-Primitive Types

// Object — labeled key-value data
const person = { name: "Chidi", age: 22 };

// Array — ordered list (arrays are objects)
const scores = [85, 90, 78];

// Function — reusable callable code
function greet(name) {
  return "Hello, " + name;
}

Primitives are copied by value. Objects, arrays, and functions are copied by reference.

typeof Operator

console.log(typeof "hello");    // "string"
console.log(typeof 42);         // "number"
console.log(typeof true);       // "boolean"
console.log(typeof undefined);  // "undefined"
console.log(typeof null);       // "object" (known quirk!)
console.log(typeof {});         // "object"

Variables & Scope

var, let, const, operators, and scope

Declaring Variables: let, const, and var

let score = 10;        // Can be reassigned
const PI = 3.14159;    // Cannot be reassigned — use for values that never change
var oldStyle = "avoid"; // Function-scoped, hoisted — prefer let/const

score = 20;            // ✅ OK — let allows reassignment
// PI = 4;             // ❌ Error — const cannot be reassigned
KeywordReassignable?ScopeWhen to Use
constNoBlockDefault choice — constants, references
letYesBlockValues that will change (counters, flags)

JavaScript Data Types

// Primitives
let name = "Amara";           // String

let isStudent = true;         // Boolean
let nothing = null;           // Null (intentional absence of value)
let notDefined;               // Undefined (declared but no value assigned)
let id = Symbol("id");        // Symbol (unique identifier)

// Reference type
let person = { name: "Chidi", age: 30 };  // Object

typeof Operator

console.log(typeof "hello");   // "string"
console.log(typeof 42);        // "number"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" (known quirk!)
console.log(typeof {});          // "object"

Arithmetic Operators

let a = 10, b = 3;

Assignment and Comparison Operators

// Comparison — always use === and !==

console.log(5 != "5");   // false
console.log(5 !== "5");  // true

// Logical operators

let canEnter = isAdult && hasTicket;  // AND
let isWeekend = day === "Sat" || day === "Sun";  // OR
let isClosed = !isOpen;                 // NOT

Template Literals

let firstName = "Ada";
let lastName = "Okafor";
let greeting = `Hello, ${firstName} ${lastName}!`;
console.log(greeting); // "Hello, Ada Okafor!"

let multiline = `
  Line one
  Line two
`;
console.log(multiline);

Type Coercion (Watch Out!)

console.log("5" + 3);    // "53"  (string concatenation)
console.log("5" - 3);    // 2     (subtraction converts to number)
console.log(true + 1);   // 2     (true becomes 1)
console.log(false + 1);  // 1     (false becomes 0)

// Always use Number() or parseInt() when converting user input
let userInput = "42";
let num = Number(userInput);
⚠️
Golden Rule

Always use === and !== for comparisons. Loose equality (==) causes subtle bugs.

JavaScript Scope

Scope refers to where variables, functions, and objects are visible and can be used in your code.

Global Scope

const appName = "TodoApp";  // global — visible everywhere

function showApp() {
  console.log(appName);  // ✅ works
}

Function Scope

function demo() {
  let secret = "hidden";
  console.log(secret);  // ✅
}
// console.log(secret);  // ❌ Error — not accessible outside

Block Scope

if (true) {
  let temp = 10;
  console.log(temp);  // ✅
}
// console.log(temp);  // ❌ Error

for (let i = 0; i < 3; i++) {
  console.log(i);  // ✅ inside loop
}
// console.log(i);  // ❌ Error — let is block-scoped
⚠️
var vs let

var ignores block scope and leaks out of if and for blocks. Always prefer let and const.

Lesson 06

Boolean Logic

True, false, comparisons, and truthy values

True, False and Beyond

Booleans represent yes/no decisions. But JavaScript also treats any value as truthy or falsy inside conditions like if (value).

Comparison Operators

5 > 3    // true   greater than
5 < 3    // false  less than
5 >= 5   // true   greater or equal
5 <= 4   // false  less or equal
5 === 5  // true   strict equal (value AND type)
5 !== 3  // true   strict not equal
5 == "5" // true   loose equal (avoid!)
5 === "5" // false strict — different types
⚠️
Golden Rule

Always use === and !==. Loose equality (==) causes subtle bugs.

Logical Operators: AND, OR, NOT

true && true   // AND → true
true && false  // AND → false
true || false  // OR  → true
false || false // OR  → false
!true          // NOT → false
!false         // NOT → true

let isAdult = age >= 18;
let hasTicket = true;
let canEnter = isAdult && hasTicket;

let isWeekend = day === "Sat" || day === "Sun";

Truthy and Falsy Values

Falsy values (only these 6 + false):

  • false
  • 0
  • "" (empty string)
  • null
  • undefined
  • NaN

Everything else is truthy — including "0", " ", [], {}.

if ("hello") {
  console.log("Truthy!");  // runs
}
if (0) {
  console.log("Never runs");
}
if ("") {
  console.log("Never runs");
}

Lesson 07

JavaScript Functions

Declarations, arrows, parameters, callbacks

Why Functions Matter

A function is a reusable block of code that performs a specific task. Instead of writing the same logic ten times, you write it once and call it whenever needed. Functions are the building blocks of maintainable, professional JavaScript.

Function Declaration

function greet(name) {
  return "Hello, " + name + "!";
}

console.log(greet("Amara")); // "Hello, Amara!"
console.log(greet("Chidi")); // "Hello, Chidi!"

Function Expression

const calculateArea = function (width, height) {
  return width * height;
};

console.log(calculateArea(5, 10)); // 50

Arrow Functions (ES6)

// Full syntax
const multiply = (a, b) => {
  return a * b;
};

// Concise — implicit return
const square = (n) => n * n;
const sayHi = () => console.log("Hi!");

console.log(multiply(3, 4)); // 12
console.log(square(5));      // 25

Parameters and Default Values

function createUser(name, role = "student", active = true) {
  return { name, role, active };
}

console.log(createUser("Ada"));
// { name: "Ada", role: "student", active: true }

console.log(createUser("Emeka", "instructor", false));
// { name: "Emeka", role: "instructor", active: false }

Return Values

function isEven(num) {
  return num % 2 === 0;
}

function findMax(a, b, c) {
  if (a >= b && a >= c) return a;
  if (b >= a && b >= c) return b;
  return c;
}

console.log(isEven(4));    // true
console.log(findMax(3, 9, 5)); // 9

Callback Functions

A callback is a function passed as an argument to another function, to be run later.

function runTwice(fn) {
  fn();
  fn();
}

runTwice(() => console.log("Hello!"));

// Real example — runs after a delay
setTimeout(function () {
  console.log("Runs after 2 seconds");
}, 2000);

Lesson 08

JavaScript Arrays

Ordered lists of any data type

What Is an Array?

An array is an ordered list of values. Use arrays when order matters — shopping cart items, quiz scores, or a list of names.

Creating Arrays

const numbers = [10, 20, 30];              // array of numbers
const names = ["Ada", "Emeka", "Chi"];     // array of strings
const mixed = [1, "hello", true, null];    // mixed data types
const matrix = [[1, 2], [3, 4]];           // array of arrays
const users = [{ name: "Ada" }, { name: "Bob" }];  // array of objects

Accessing Elements

console.log(numbers[0]);   // 10 — first element
console.log(numbers[2]);   // 30 — third element
console.log(numbers.length); // 3 — number of items

Modifying Elements

numbers[1] = 99;
console.log(numbers);  // [10, 99, 30]

Array Methods: push, pop, shift, unshift

const fruits = ["Apple", "Banana"];

View Full Code 
let todos = [];
let running = true;

alert("Welcome to Todo App!\nCommands: add, list, delete, quit");

while (running) {
  let command = prompt("Enter command (add / list / delete / quit):");

  if (command === null) {
    alert("Goodbye!");
    break;
  }

  command = command.toLowerCase().trim();

  if (command === "add") {
    let task = prompt("Enter task to add:");
    if (task && task.trim() !== "") {
      todos.push(task.trim());
      alert("Task added!");
      console.log("Added:", task);
    } else {
      alert("Task cannot be empty.");
    }

  } else if (command === "list") {
    if (todos.length === 0) {
      alert("No tasks yet.");
      console.log("Todo list is empty.");
    } else {
      let list = "Your Tasks:\n";
      for (let i = 0; i < todos.length; i++) {
        list += (i + 1) + ". " + todos[i] + "\n";
      }
      alert(list);
      console.log(list);
    }

  } else if (command === "delete") {
    if (todos.length === 0) {
      alert("Nothing to delete.");
    } else {
      let list = "Which task to delete?\n";
      for (let i = 0; i < todos.length; i++) {
        list += (i + 1) + ". " + todos[i] + "\n";
      }
      let index = prompt(list + "Enter number:");
      let num = Number(index);

      if (num >= 1 && num <= todos.length) {
        let removed = todos.splice(num - 1, 1);
        alert("Deleted: " + removed[0]);
        console.log("Deleted:", removed[0]);
      } else {
        alert("Invalid number.");
      }
    }

  } else if (command === "quit") {
    alert("Thanks for using Todo App!");
    running = false;

  } else {
    alert("Unknown command. Use: add, list, delete, quit");
  }
}

Concepts Used

  • Variables: todos array, running flag
  • Arrays: push, splice, length, indexing
  • Conditionals: if / else if / else chains
  • Loops: while for the app, for to display list
  • I/O: prompt, alert, console.log
  • Strings: trim, toLowerCase, concatenation

for (let i = 0; i < 5; i++) {

Document and Browser Object Models

Interacting with the webpage and browser

What is the DOM?

The Document Object Model (DOM) is a programming interface that represents the HTML document as a tree of objects. JavaScript can access and manipulate these objects to change content, structure, and styling dynamically.

Selecting Elements

// Select by ID
let heading = document.getElementById("main-title");

// Select by class name (returns HTMLCollection)
let cards = document.getElementsByClassName("card");

// Select by tag name
let paragraphs = document.getElementsByTagName("p");

// Select using CSS selectors (modern approach)
let firstCard = document.querySelector(".card");      // First match
let allCards = document.querySelectorAll(".card");    // All matches

Manipulating Content

// Change text content
document.getElementById("demo").textContent = "New text";

// Change HTML content
document.getElementById("demo").innerHTML = "<strong>Bold text</strong>";

// Change attributes
let img = document.querySelector("img");
img.src = "new-image.jpg";
img.alt = "New description";

// Change styles
let box = document.querySelector(".box");
box.style.color = "red";
box.style.backgroundColor = "yellow";
box.style.fontSize = "20px";

Creating and Removing Elements

// Create new element
let newParagraph = document.createElement("p");
newParagraph.textContent = "This is a new paragraph";
document.body.appendChild(newParagraph);

// Remove element
let oldElement = document.getElementById("old");
oldElement.remove();

// Or remove child from parent
let parent = document.getElementById("container");
let child = document.getElementById("item");
parent.removeChild(child);

The Browser Object Model (BOM)

The BOM allows JavaScript to interact with the browser itself.

// Window object (global object in browsers)
window.alert("Hello!");

window.prompt("Enter your name:");

// Get window dimensions

// Timers

  console.log("Runs after 2 seconds");

  console.log("Runs every 3 seconds");

Method Description
getElementById() Select element by ID attribute
getElementsByClassName() Select elements by class name
querySelector() Select first element matching CSS selector
querySelectorAll() Select all elements matching CSS selector

Event-driven Programming

Responding to user actions and browser events

What Is Event-driven Programming?

Modern websites don't just display static content — they react to what users do. Event-driven programming means your code waits for something to happen (a click, key press, form submit, page load) and then runs a function in response. This is the foundation of every interactive web application.

Common Browser Events

EventFires When…
clickUser clicks an element
dblclickUser double-clicks an element
mouseenter / mouseleaveMouse enters or leaves an element
keydown / keyupUser presses or releases a key
inputValue of an input field changes
submitA form is submitted
loadPage or resource finishes loading
DOMContentLoadedHTML is fully parsed (before images load)
scrollUser scrolls the page or an element

Inline Event Handlers (Avoid in Production)

<button onclick="alert('Clicked!')">Click</button>

Inline handlers mix HTML and JavaScript. They work for learning, but professional code separates concerns.

addEventListener — The Professional Approach

// Select the element
let btn = document.querySelector("#myButton");

// Attach an event listener
btn.addEventListener("click", function () {
  console.log("Button was clicked!");
});

// Arrow function syntax (modern)
btn.addEventListener("click", () => {
  alert("Hello from the button!");
});

The Event Object

When an event fires, the browser passes an event object containing useful information.

document.querySelector("input").addEventListener("keydown", function (event) {
  console.log("Key pressed:", event.key);
  console.log("Target element:", event.target);
});

document.querySelector("a").addEventListener("click", function (event) {
  event.preventDefault(); // Stops the link from navigating
  console.log("Link click intercepted");
});

Event Propagation: Bubbling and Capturing

Events travel through the DOM in two phases. In the bubbling phase, the event starts at the innermost element and moves up to document. In capturing, it travels downward first. By default, listeners run during the bubbling phase. 

let parent = document.querySelector(".parent");
let child = document.querySelector(".child");

parent.addEventListener("click", () => console.log("Parent clicked"));
child.addEventListener("click", () => console.log("Child clicked"));
// Clicking child logs: "Child clicked" then "Parent clicked"

child.addEventListener("click", (e) => {
  e.stopPropagation(); // Prevents event from reaching parent
});

Multiple Listeners on One Element

let card = document.querySelector(".card");

card.addEventListener("mouseenter", () => {
  card.style.transform = "scale(1.02)";
});

card.addEventListener("mouseleave", () => {
  card.style.transform = "scale(1)";
});

card.addEventListener("click", () => {
  card.classList.toggle("selected");
});

Waiting for the DOM to Load

document.addEventListener("DOMContentLoaded", function () {
  // Safe to query and manipulate elements here
  let heading = document.querySelector("h1");
  heading.textContent = "Page is ready!";
});

Instructor Tip

Always use addEventListener instead of inline onclick attributes. You can attach multiple handlers, remove them later, and keep your HTML clean.

Objects & Form Validation

Objects, JSON, and validating forms

What Is an Object?

An object stores related data as key-value pairs (properties). Think of it as a labeled container — a user profile, a product, or a car registration all map naturally to objects. 

const student = {
  firstName: "Amara",
  lastName: "Bello",
  age: 22,
  isEnrolled: true,
  courses: ["HTML", "CSS", "JavaScript"]
};

// Access properties
console.log(student.firstName);      // Dot notation
console.log(student["lastName"]);    // Bracket notation

// Add or update properties
student.email = "amara@school.com";
student.age = 23;

// Delete a property
delete student.isEnrolled;

Object Methods

const calculator = {
  value: 0,
  add(n) {
    this.value += n;
    return this;
  },
  reset() {
    this.value = 0;
  }
};

calculator.add(5).add(3);
console.log(calculator.value); // 8

Looping Through Objects

const user = { name: "Chidi", role: "developer", level: 2 };

// for...in — object properties
for (let key in user) {
  console.log(key + ": " + user[key]);
}

JSON — Storing and Sharing Data

const product = { id: 1, name: "Laptop", price: 450000 };

// Object → JSON string (for APIs, localStorage)
const jsonString = JSON.stringify(product);
console.log(jsonString); // '{"id":1,"name":"Laptop","price":450000}'

// JSON string → Object
const parsed = JSON.parse(jsonString);
console.log(parsed.name); // "Laptop"

Instructor Tip

Use objects when you need labeled fields (a person, a settings config). Use arrays when you need an ordered list (search results, menu items).

Form Validation — A Real-world Application

Validating forms before submission is one of the most common uses of JavaScript in frontend development.

<form id="signupForm">
  <input type="text" id="username" placeholder="Username">
  <input type="email" id="email" placeholder="Email">
  <input type="password" id="password" placeholder="Password">
  <p id="errorMsg" style="color:red"></p>
  <button type="submit">Sign Up</button>
</form>
function validateEmail(email) {
  return email.includes("@") && email.includes(".");
}

function validatePassword(password) {
  return password.length >= 8;
}

function validateUsername(username) {
  return username.trim().length >= 3;
}

function showError(message) {
  document.getElementById("errorMsg").textContent = message;
}

document.getElementById("signupForm").addEventListener("submit", function (event) {
  event.preventDefault();

  const username = document.getElementById("username").value;
  const email = document.getElementById("email").value;
  const password = document.getElementById("password").value;

  if (!validateUsername(username)) {
    showError("Username must be at least 3 characters.");
    return;
  }
  if (!validateEmail(email)) {
    showError("Please enter a valid email address.");
    return;
  }
  if (!validatePassword(password)) {
    showError("Password must be at least 8 characters.");
    return;
  }

  showError("");
  alert("Account created successfully!");
});
⚠️
Security Note

Never rely on JavaScript validation alone. Always validate on the server too.
